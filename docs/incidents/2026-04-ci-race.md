# 2026-04-11 — CI-only race in delivery-times.e2e.spec.ts

## Symptom

Local `npm test` passed 20/20 runs, but the same suite failed in CI with a one-line race in `delivery-times.e2e.spec.ts`. The failure was timing-dependent and required CI's environment shape (clock, container resource limits, network jitter) to surface.

## Root cause

A test was assuming a specific ordering between two scheduled events that local-machine timing happened to satisfy but CI's slower scheduling did not. The race was masked locally because the dev machine's faster clock + lighter container load happened to interleave the events in the order the test expected.

## Resolution

Fix the test to assert on the underlying invariant (which event must complete before which) rather than on observed ordering. Then run Act locally to reproduce CI's environment before pushing.

## Standing rule

See [docs/ci-with-act.md](../ci-with-act.md): run the full pre-commit stack (or the same commands manually) before pushing changes that touch CI, workflows, tests, test helpers, vitest/playwright config, or core build tooling. Local `npm test` alone is not sufficient signal for those changes.
